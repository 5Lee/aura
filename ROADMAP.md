# Aura 项目长期开发路线图

## 🎯 项目愿景

打造一个专业、易用的 AI 提示词管理平台，帮助 AI 从业者和爱好者高效管理、分享和优化他们的提示词资产。

---

## 📊 当前状态 (2026-03-02)

### 已完成功能 (20/20) ✅
- 用户认证系统
- 提示词 CRUD
- 分类标签管理
- 搜索功能
- 收藏系统
- 公开浏览
- 响应式设计
- 深色模式
- Docker 部署

**进度**: MVP (最小可行产品) 完成

### 🤖 Agent Harness 系统
基于 Anthropic 的 "Effective Harnesses for Long-Running Agents" 最佳实践：
- `init.sh` - 开发环境启动脚本
- `feature_list_phase1.json` - Phase 1 功能列表 (JSON 格式)
- `CLAUDE_PROGRESS.md` - 开发进度记录
- `AGENT_SESSION_GUIDE.md` - Agent Session 工作指南
- `.mcp.json` - Playwright MCP 配置 (用于 E2E 测试和浏览器调试)

## 🔄 持续迭代流程

### 日常开发节奏
```
周一: 规划会议 - 确定本周任务
周二-周四: 开发实施
周五: 代码审查 + 测试 + 部署
```

### 两周迭代周期
```
Sprint 1 (Week 1): 开发新功能
Sprint 2 (Week 2): Bug 修复 + 优化 + 发布
```

### 月度回顾
- 功能使用数据分析
- 用户反馈收集
- 性能指标评估
- 下月计划调整

## 📋 任务优先级决策树

```
是否影响核心功能？
├─ 是 → 是否是安全/数据问题？
│       ├─ 是 → P0 (立即)
│       └─ 否 → P1 (本周)
└─ 否 → 是否是用户强烈需求？
        ├─ 是 → P2 (本月)
        └─ 否 → P3 ( backlog )
```
